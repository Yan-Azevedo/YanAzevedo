## 🏗️ How I Work | Como eu trabalho

I follow a structured approach to avoid "AI for the sake of AI":
Sigo uma abordagem estruturada para evitar “IA por fazer”:

1. **Diagnosis before solution** | Diagnóstico antes da solução
2. **Process understanding** | Entendimento do processo
3. **High-impact use case definition** | Definição de casos de alto impacto
4. **Guided implementation (no hype-driven builds)** | Construção guiada (sem hype)
5. **Clear delivery with documentation and next steps** | Entrega com documentação e próximos passos claros
